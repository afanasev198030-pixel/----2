
# Правила AI‑заполнения ДТ по графам (v5.0)

Источник требований: Решение КТС от 20.05.2010 № 257 «О форме декларации на товары и порядке её заполнения» (в действующей редакции) + ТЗ v3.0.

Цель этого документа — описать **правила**, по которым AI заполняет ДТ (декларацию на товары) **на основе документов**, с контролем источников, проверками и флагами "нужна ручная проверка".

---

## 1) Общие принципы (как AI должен работать)

- **Только evidence‑based**: AI заполняет графу, только если может сослаться на документ‑источник (контракт/инвойс/PL/транспорт/сертификаты). Если данных нет — оставляет пусто и ставит "нужно уточнить".
- **Приоритет источников (если конфликтуют)**:
  1) **Подтверждающие/разрешительные** документы (сертификат происхождения, решения ФТС, лицензии/квоты, ДТС‑1 и т.п.)
  2) **Контракт + доп. соглашения**
  3) **Инвойс**
  4) **Packing List / спецификация**
  5) **Транспортные документы**
  6) **Внутренние справочники/профиль компании** (ИНН/КПП, реквизиты)
  7) **Модельные догадки** (разрешены только как "кандидаты", не как финальное значение для критичных граф)
- **Нормализация**:
  - страны: ISO 3166‑1 alpha‑2 (CN, RU, DE…) — в соответствии с **Классификатором стран мира**
  - валюта: ISO 4217 (USD, EUR, CNY…) — в соответствии с **Классификатором валют**
  - ТН ВЭД: строго 10 цифр (01–97 в первых двух)
  - числа: разделители пробел/запятая → привести к float/Decimal
- **Валидации и "красные флаги"**: если value не проходит формат/диапазон/связь между графами — AI не "прибивает гвоздями", а просит подтверждение.
- **Сопровождающая мета‑информация (рекомендовано)**: для каждого значения хранить:
  - `value`
  - `sources[]` (какой документ/страница/фрагмент)
  - `confidence` (0..1)
  - `notes` (почему выбран именно этот источник / что конфликтует)

---

## 2) Документы (минимальный набор и классификация)

По ТЗ обязательные документы для импорта/экспорта:
- Контракт/договор (+ доп. соглашения)
- Инвойс на товары
- Упаковочные листы
- Спецификация (общая на все товары из заказа)
- Транспортные документы (CMR / B/L / AWB / CIM)
- Транспортный инвойс
- Сертификат происхождения (СТ‑1 / Form A / EUR.1), если необходимо
- Application (заявка)

AI должен:
- сначала **классифицировать** каждый файл по типу (invoice/contract/packing/transport/origin_certificate/other),
- затем извлечь факты в единый "паспорт сделки",
- затем заполнить графы по правилам ниже.

## 3) Правила заполнения граф (шапка ДТ)

### Графа 1 — Декларация (тип/процедура)
- **Классификатор**: Классификатор видов таможенных процедур.
- **Три подраздела**:
  - **Подраздел 1**: `ИМ` (импорт) или `ЭК` (экспорт).
  - **Подраздел 2**: двузначный цифровой код заявляемой таможенной процедуры (по классификатору). Примеры: `40` — выпуск для внутреннего потребления, `70` — таможенный склад, `51` — переработка на таможенной территории.
  - **Подраздел 3**: `ЭД` — только при подаче ДТ в виде электронного документа.
- **Источники**: пользователь выбирает тип при создании декларации.
- **Правило**: заполнять код процедуры в соответствии с выбранным типом. Пример: `ИМ | 40 | ЭД`.
- **Если не найдено**: предложить ИМ40 для импорта с флагом "проверьте".

### Графа 2 — Отправитель/Экспортёр
- **Источники** (в порядке приоритета):
  1. **Транспортный документ** (AWB / CMR / B/L) — поле «Shipper» / «Отправитель» + адрес. Основной источник.
  2. **Заявка** (Application) — поле «Отправитель». Резервный источник.
- **Заполнить** (в зависимости от типа лица):
  - **Организация**: краткое наименование + место нахождения (страна по **Классификатору стран мира**, адрес).
  - **Физическое лицо**: ФИО + место жительства (страна, адрес).
- **Правый верхний угол (после «N»)** — для лиц государств-членов ЕАЭС:
  - РФ: ИНН и через «/» КПП (для юрлиц); если обособленное подразделение — КПП подразделения.
  - РБ: учётный номер плательщика (УНП).
  - РК: БИН для организаций, ИИН для физлиц.
- **Нижняя часть графы** (для лиц государств-членов ЕАЭС):
  - РФ: ОГРН или ОГРНИП.
  - РК: ИТН по Классификатору формирования ИТН.
- **Графа НЕ заполняется**, если:
  - При завершении действия предшествующей процедуры товар не перемещался через границу.
  - Декларируется наличная валюта (выручка) на бортах воздушных судов и т.п.
  - Декларируются товары, приобретённые в розничной торговле за пределами ЕАЭС.
- **Если отправитель — иностранное лицо или лицо иного государства-члена ЕАЭС**: правый верхний угол и нижняя часть графы не заполняются.
- **AI-правило**: определить тип лица (организация / физлицо) по наличию организационно-правовой формы (LLC, Co., Ltd., GmbH, ООО и т.п.). При неоднозначности — заполнить как организацию + флаг «проверьте тип отправителя».
- **Валидация**: страна — обычно не RU (для импорта); если RU — предупреждение.

### Графа 3 — Формуляры (листы ДТ)
- **Формат**: `X/Y` (X — порядковый номер текущего листа, Y — общее количество листов ДТ).
- **Источники**: инвойс на товары (количество позиций).
- **Правило расчёта**:
  - Основной лист — 1 товар; каждый добавочный лист — до 3 товаров.
  - `total_sheets = 1 + ceil((total_items - 1) / 3)` при `total_items > 1`; при 1 товаре `total_sheets = 1`.
  - Заполняется на **каждом листе**. Пример: 4 товара → `1/3`, `2/3`, `3/3`.

### Графа 4 — Отгрузочные спецификации
- **Формат**: `количество_спецификаций / общее_количество_листов`.
  - Пример: 2 спецификации по 3 листа каждая → `2/6`.
- **Источники**: отгрузочные спецификации к контракту.
- **Правило**: заполняется **только если есть** отгрузочные спецификации, перечни или подобные документы. В остальных случаях — пусто.

### Графа 5 — Всего товаров
- **Правило**: `total_items_count` = количество товарных **наименований (позиций)**, а не штук.
- **Источники**: инвойс на товары + Packing List (PL).
  - **⚠️ Важно**: использовать именно **инвойс на товары**, а не транспортный инвойс.
  - Считать количество строк товарных позиций (line items).
- **Контроль**: должно соответствовать количеству заполненных граф 31 основного и добавочных листов. При расхождении с PL — флаг.

### Графа 6 — Количество мест
- **Источники**: packing list (итог мест), транспортные накладные.
- **Правило**: брать total packages из PL; при конфликте с транспортным документом — обе цифры + флаг.
- **Особые случаи**:
  - Перевозка насыпом, навалом, наливом (нет грузовых мест в документах) → указать **«0»**.
  - Не заполняется, если при завершении действия предшествующей процедуры не было международной перевозки.

### Графа 7 — Справочный номер
- **Классификатор**: Классификатор особенностей таможенного декларирования товаров.
- **Правило**: заполняется только если есть применимый код. Основные коды:
  - `ПТД` — предварительное декларирование.
  - `ОКТ` — декларирование товара в несобранном/разобранном виде (отдельные компоненты).
  - `ЗПК` — декларирование последней партии компонентов разобранного товара.
  - `ПКТ` / `ППК` — предварительное декларирование компонентов / последнего компонента (РФ).
  - `ВТД` — временная ДТ (временное периодическое декларирование).
  - `ПВД` — полная ДТ при временном периодическом декларировании.
  - `НВТ` — товары, незаконно перемещённые через таможенную границу.
- **Если особенностей нет** — оставить пустым (не заполнять «по умолчанию»).
- **AI-правило**: подсветить пользователю с пояснением: «Укажите код особенности или оставьте пустым».

### Графа 8 — Получатель
- **Классификатор**: Классификатор стран мира (для адреса).
- **Заполнить**: наименование, адрес, ИНН/КПП, ОГРН.
- **Единственный источник**: **трёхсторонний договор**. Другие источники (профиль компании, контракт, инвойс) НЕ используются.
- **Логика заполнения**:
  1. Проверить наличие трёхстороннего договора среди загруженных документов.
  2. **Если трёхсторонний договор НЕ найден** → заполнить **«СМ. ГРАФУ 14 ДТ»** (получатель = декларант).
  3. **Если трёхсторонний договор найден** → извлечь данные получателя из договора (раздел «Получатель» / «Consignee» / «Receiver»): наименование, адрес, ИНН/КПП, ОГРН.
- **Правый верхний угол (после «N»)** — аналогично Графе 2:
  - РФ: ИНН/КПП.
  - РБ: УНП.
  - РК: БИН/ИИН.
- **Нижняя часть**:
  - РФ: ОГРН/ОГРНИП.
  - РК: ИТН.
- **Не заполняется**: при декларировании наличной валюты на бортах ВС и ж/д транспорте.
- **Приоритет языка**: русский приоритетнее английского.

### Графа 9 — Лицо, ответственное за финансовое урегулирование
- **Единственный источник**: **трёхсторонний договор**. Другие источники (доверенность, договор с брокером) НЕ используются.
- **Логика заполнения**:
  1. Проверить наличие трёхстороннего договора среди загруженных документов.
  2. **Если трёхсторонний договор НЕ найден** → заполнить **«СМ. ГРАФУ 14 ДТ»** (финансово ответственное лицо = декларант).
  3. **Если трёхсторонний договор найден** → извлечь данные лица, ответственного за финансовое урегулирование, из договора (раздел «Лицо, ответственное за финансовое урегулирование» / «Financial responsible party»): наименование, адрес, ИНН/КПП, ОГРН.
- **Перечень лиц** (по инструкции, одно из следующих):
  1. Лицо государства-члена ЕАЭС, заключившее договор (контракт) по сделке, по которой товары ввезены.
  2. Иностранное лицо, имеющее право распоряжаться товарами не в рамках сделки с лицом государства-члена ЕАЭС.
  3. Собственник товаров при односторонней сделке.
  4. Лицо, приобретшее имущественные права на товары (в т.ч. при завершении предшествующей процедуры).
  5. Лицо, ввозящее наличную валюту.
- **Заполнить**: по формату аналогичному графам 2/8 (наименование, адрес, ИНН/КПП, ОГРН).
- **Приоритет языка**: русский приоритетнее английского.

## 4) Правила по странам (графы 10–17)

### Графа 10 — (не заполняется декларантом)
- Предназначена для иных целей. Пропустить.

### Графа 11 — Торгующая страна
- **Классификатор**: Классификатор стран мира.
- **Два подраздела**:
  - **Подраздел 1**: код страны контрагента по сделке (ISO alpha-2 или цифровой по классификатору).
  - **Подраздел 2**: для РБ — трёхзначный код административно-территориального деления (по Классификатору адм.-терр. деления стран РБ). Для РФ и РК — не заполняется.
- **Источники** (в порядке приоритета):
  1. Контракт — страна из реквизитов продавца/поставщика.
  2. Сертификат происхождения — страна контрагента.
  3. Инвойс на товары — страна продавца.
- **Особые случаи**:
  - При отсутствии сделки (РБ) — код `00`.
  - Не заполняется при декларировании наличной валюты на бортах ВС.

### Графа 12 — Общая таможенная стоимость
- **Правило**: сумма значений из граф 45 основного и добавочных листов. Округление до 2 знаков (для РФ, РК, КР); для Армении — до целого числа.
- **Не заполняется**: при процедурах таможенного склада, уничтожения, отказа в пользу государства, специальной таможенной процедуре; при декларировании валюты и ценных бумаг.

### Графа 13 — Резервная
- Не заполняется участниками ВЭД. Пропустить.

### Графа 14 — Декларант
- **Процедура**: импорт (ИМ). Информация указывается **на русском языке**.
- **Заполнить**: краткое наименование, адрес (место нахождения), ИНН/КПП, ОГРН.
- **Источники** (в порядке приоритета):
  1. **Контракт** — основной источник. Извлекать из раздела «Покупатель» / «Buyer»: наименование, адрес, ИНН/КПП, ОГРН. Предпочитать русский язык.
  2. **Инвойс на товары** — резервный источник, если контракт не содержит данных. Поле «Покупатель» / «Buyer». Предпочитать русский язык.
  3. **Профиль компании в системе** — fallback, если контракт и инвойс не содержат данных. Поля: наименование, адрес, ИНН, КПП, ОГРН.
- **Логика заполнения**:
  - Каждое поле (наименование, адрес, ИНН, КПП, ОГРН) проверяется отдельно: берётся первое непустое значение в порядке приоритета (контракт → инвойс → профиль компании).
  - Если декларант — уполномоченный экономический оператор → дополнительно номер свидетельства о включении в реестр УЭО.
- **Приоритет языка**: русский. Если данные в контракте/инвойсе только на английском — использовать их, но при наличии русскоязычного варианта — предпочитать русский.

### Графа 15 / 15А — Страна отправления
- **Классификатор**: Классификатор стран мира.
- **Графа 15**: краткое название страны отправления.
- **Графа 15А**: подраздел «a» — код страны (ISO alpha-2). Подраздел «b» — для РБ (код адм.-терр. деления); для РФ — не заполняется.
- **Источники** (в порядке приоритета):
  1. Транспортные документы AWB / CMR — поле «Departure» / «From».
  2. Транспортный инвойс — поле «Departure».
- **Определение**: страна, из которой **начата** международная перевозка (не страна происхождения). При нескольких транспортных документах — первая страна, где товар принят к перевозке.
- **Не заполняется**: при отсутствии международной перевозки (завершение предшествующей процедуры), при декларировании наличной валюты.

### Графа 16 — Страна происхождения
- **Классификатор**: Классификатор стран мира.
- **Источники**: сертификат происхождения, декларация о соответствии (поле «страна изготовителя»).
- **Правила заполнения**:
  - Указывается краткое название страны (группы стран, таможенного союза стран, региона или части страны).
  - Если документы указывают на **Европейский союз** без конкретной страны → **«ЕВРОСОЮЗ»**.
  - Если в ДТ несколько товаров из **разных стран** ИЛИ страна хотя бы одного товара неизвестна → **«РАЗНЫЕ»**.
  - Если страна происхождения **всех** товаров неизвестна → **«НЕИЗВЕСТНО»** (не пустое поле!).
  - Не заполняется при декларировании наличной валюты.

### Графа 17 / 17А — Страна назначения
- **Классификатор**: Классификатор стран мира.
- **Источники**: транспортные документы (поле «страна прибытия» / «destination» / «consignee country»).
- **Графа 17**: краткое название страны назначения (где товары будут потребляться/использоваться).
- **Графа 17А**:
  - Подраздел «a» — код страны назначения (ISO alpha-2).
  - Подраздел «b» — для РБ (код адм.-терр. деления); для РФ — не заполняется.
- **Значение «НЕИЗВЕСТНА»** — только в случаях:
  1. Беспошлинная торговля (вывоз).
  2. Неполная / периодическая ДТ (сведения будут дополнены позднее).
  3. Временная (дополнительная временная) ДТ.
- В остальных случаях «НЕИЗВЕСТНА» не используется; если страна неизвестна — указать страну доставки.
- **Не заполняется**: при декларировании наличной валюты.

## 5) Правила по транспорту (графы 18–30)

### Графа 18 — Идентификация и страна регистрации ТС при отправлении
- **Классификаторы**: Классификатор стран мира (код страны ТС), Классификатор видов транспорта и транспортировки товаров.
- **Источники**: транспортные документы AWB / CMR / B/L / ЖД-накладная.
- **Подраздел 1**: количество ТС и после двоеточия через «;» без пробела:
  - **Авто**: регистрационные номера. Состав ТС: тягач + через «/» прицепы. Пример: `2:С225ВН163/АВ1971163`.
  - **ЖД**: номера вагонов (платформ, цистерн).
  - **Море/река**: наименования судов.
  - **Авиа**: номера рейсов.
  - **Трубопровод/ЛЭП**: способ транспортировки (газопровод, нефтепровод, линии электропередачи).
- **Подраздел 2**: код страны регистрации ТС (ISO alpha-2).
  - Состав ТС → код страны тягача (приводящего в движение).
  - Страна неизвестна → `00`.
  - Несколько ТС из разных стран → `99`.
  - При ЖД и трубопроводе/ЛЭП → **не заполняется**.
- **Не заполняется**: при отсутствии международной перевозки, при МПО, в РБ.
- **Может не заполняться**: при предварительном / периодическом декларировании (с последующим дополнением).

### Графа 19 — Контейнер
- **Правило**: `1` — да (перевозка в контейнере), `0` — нет (включая случай, когда неизвестно).
- **Источники**: транспортные документы AWB / CMR / B/L — наличие номера контейнера.

### Графа 20 — Условия поставки
- **Классификатор**: Классификатор условий поставки.
- **Три подраздела**:
  - **Подраздел 1**: НЕ заполняется.
  - **Подраздел 2**: код условия поставки (Инкотермс) + наименование географического пункта. Пример: `CIF МОСКВА`, `FOB SHANGHAI`.
  - **Подраздел 3**: НЕ заполняется (в РБ — двузначный код вида поставки).
- **Если условия разные** для товаров в одной ДТ → **«РАЗНЫЕ»** в подразделе 2. Расшифровка — в Графе 31 (подраздел 5) для каждого товара.
- **Источники** (в порядке приоритета):
  1. Заявка (application) — если содержит условия поставки.
  2. Договор / контракт — раздел «Delivery Terms» / «Базис поставки».
- **Примечание**: при расхождении между заявкой и договором — приоритет заявки, предупреждение для проверки.

### Графа 21 — Идентификация и страна регистрации активного ТС на границе
- **Классификаторы**: Классификатор стран мира, Классификатор видов транспорта и транспортировки товаров.
- **Источники**: транспортный документ AWB / CMR / B/L (**не транспортный инвойс**).
- **Подраздел 1**: количество ТС + через «:» и «;» без пробела:
  - **AWB** (авиа) → **номер рейса** (flight number). **Не путать с номером AWB!**
  - **CMR** (авто) → **государственный регистрационный номер** грузового автомобиля (тягача).
  - **B/L** (море/река) → **название судна** (vessel name).
  - **ЖД** → номер поезда или локомотива.
  - **Трубопровод / ЛЭП** → способ транспортировки.
- **Подраздел 2**: код страны регистрации ТС (ISO alpha-2). Правила: тягач, `00` если неизвестна, `99` если несколько стран, не заполняется при ЖД и трубопроводе/ЛЭП.
- **Не заполняется**: при отсутствии международной перевозки, при МПО, при ЖД, в РБ.

### Графа 22 — Валюта и общая фактурная стоимость
- **Классификатор**: Классификатор валют (ISO 4217).
- **Подраздел 1**: буквенный код валюты цены контракта (для РФ, РК, Армении, КР) или валюты платежа (для РБ).
  - Источник кода валюты → **только договор/контракт** (не инвойс).
  - Искать фразы: «Валютой Контракта являются…», «Contract currency is…», «Payment currency:…», «расчёты производятся в…».
  - При нескольких валютах → код **преобладающей** валюты.
- **Подраздел 2**: общая стоимость = `sum(items[].graph_42)`. Округление до 2 знаков.
- **Источники**:
  - Код валюты → **договор/контракт купли-продажи** (единственный источник, не инвойс).
  - Сумма → рассчитывается как `sum(items[].graph_42)`. Для сверки — итог инвойса на товары.
- **Контроль**: рассчитанная сумма должна совпадать с итогом инвойса на товары. При расхождении → флаг.

### Графа 23 — Курс валюты
- **Правило**: курс иностранной валюты (из Графы 22) к рублю, установленный ЦБ РФ на **дату регистрации ДТ**.
- Заполняется, если требуется пересчёт иностранной валюты для определения таможенной стоимости и/или исчисления платежей.

### Графа 24 — Характер сделки
- **Классификаторы**:
  - Классификатор характера сделки (подраздел 1).
  - Классификатор особенности сделки (подраздел 2).
- **Три подраздела**:
  - **Подраздел 1**: **трёхзначный** цифровой код характера сделки. Примеры: `010` — купля-продажа, `020` — бартер, `030` — безвозмездная поставка.
  - **Подраздел 2**: **двузначный** цифровой код особенности сделки. Пример: `01` — без особенностей.
  - **Подраздел 3**: НЕ заполняется.
- **Источники**: контракт / договор.
- **Если не найдено**: дефолт `010` (купля-продажа) + `01` + флаг "проверьте".

### Графа 25 — Вид транспорта на границе
- **Классификатор**: Классификатор видов транспорта и транспортировки товаров.
- **Источники**: транспортные документы AWB / CMR / B/L (определяется по виду документа).
- **Два подраздела**:
  - **Подраздел 1**: код вида ТС из Графы 21. Основные коды: `10` — морской, `20` — ЖД, `30` — автомобильный, `40` — воздушный, `50` — почтовые отправления, `80` — внутренний водный, `90` — прочее/пеший.
  - **Подраздел 2**: НЕ заполняется.
- **AI-правило**: определить тип по виду документа: AWB → `40`, CMR → `30`, B/L → `10`.
- **Не заполняется**: при завершении действия предшествующей процедуры без международной перевозки; при помещении товаров Союза под процедуру беспошлинной торговли.

### Графа 26 — Вид транспорта внутри страны
- **Классификатор**: Классификатор видов транспорта и транспортировки товаров.
- **Источники**: определяется на основании Графы 18 (вид ТС при отправлении).
- **Два подраздела**:
  - **Подраздел 1**: код вида ТС, сведения о котором указаны в **Графе 18**. При МПО — код почтового транспорта.
  - **Подраздел 2**: НЕ заполняется.
- **Не заполняется**: при отсутствии международной перевозки; в РБ.
- **⚠️ Важно**: графа заполняется (не путать с «не заполняется» — ранее содержалась ошибка).

### Графа 27 — Место погрузки/разгрузки
- **Правило**: не заполняется в ДТ (используется в транзитной декларации).

### Графа 28 — Финансовые и банковские сведения
- **Правило**: не заполняется в ДТ.

### Графа 29 — Таможня на границе
- **Классификатор**: Классификаторы таможенных органов государств-членов ЕАЭС.
- **Формат** (зависит от страны ввоза):
  - Через РФ: 8-значный код (пример: `10206040`).
  - Через РБ/РК/КР: `ZZZ XXXXX` (3-значный код страны + 5-значный код ТО). Пример: `11209103`.
  - Через Армению: `ZZZ 000 XX` (код Армении + `000` + 2-значный код ТО).
- **Несколько пунктов пропуска**: указать все коды.
- **Не заполняется**: в РБ; при трубопроводе/ЛЭП.
- **AI-правило**: определить место прибытия из транспортных документов → сопоставить с классификатором.

### Графа 30 — Местонахождение товаров
- **Классификаторы**: Классификатор мест нахождения товаров, Классификаторы таможенных органов.
- **Заполнить** (через соответствующие реквизиты):
  - Код места нахождения товаров.
  - Код таможенного органа, в регионе деятельности которого находится место.
  - Номер свидетельства (СВХ, УЭО) или адрес / наименование ж/д станции / порта / номер ЗТК.
- **Источники**: транспортные документы (место доставки / place of delivery).
- **Не заполняется**: при выпуске товаров до подачи ДТ (ст. 120 ТК ЕАЭС), при некоторых особенностях декларирования.
- **Если информация не найдена**: пусто + `warning` «Укажите место нахождения товаров».

## 6) Правила по товарным позициям (графы 31–46, для каждой позиции)

> ### ⚠️ Ключевое правило раздела: данные — только по конкретному товару
>
> Графы 31–46 заполняются **отдельно для каждой товарной позиции**. Веса (гр. 35, 38), цена (гр. 42), таможенная стоимость (гр. 45), статистическая стоимость (гр. 46), количество и упаковка (гр. 31, 41) — только по данному наименованию.

> **Важно о спецификации**: Спецификация — **вспомогательный** документ. Список товарных позиций → из **инвойса на товары** (приоритет) или PL. Спецификация — только для **перекрёстной сверки**.

### Графа 31 — Грузовые места и описание товаров
- **Классификаторы**: Классификатор видов груза, упаковки и упаковочных материалов; Классификатор дополнительных характеристик и параметров.
- **Структура** (нумерованные подразделы, каждый с новой строки):
  - **1. Наименование товара**: фирменное/коммерческое наименование, сведения о производителе (при наличии), товарные знаки, марки, модели, артикулы, сорта, стандарты, технические и коммерческие характеристики, количественный и качественный состав. Также: характеристики и параметры в единицах измерения (отличных от основной/дополнительной) по Классификатору допхарактеристик.
    - Если товар содержит ОИС из таможенного реестра → дополнительно регистрационный номер ОИС.
  - **2. Грузовые места и упаковка** (из PL): общее количество мест, занятых товаром. Если частично → `(X часть места)`. Коды видов упаковки по Классификатору через «-» количество. Для насыпных/навалочных/наливных → код груза без упаковки. Если на поддонах → сведения о поддонах + через «/» код поддона.
  - **2.1. Потребительская/индивидуальная тара**: коды видов тары через запятую по Классификатору.
  - **3. Контейнеры** (если перевозка в контейнерах): тип контейнера по Классификатору, количество, через двоеточие номера. Если товар не занимает весь контейнер → после номера `часть`.
  - **4. Акцизные/специальные марки**: серия, номера, количество. Подряд идущие номера — через «–» (диапазон).
  - **5. Условия поставки** (если в Графе 20 указано «РАЗНЫЕ»): код условия поставки через «/» для данного товара + через «–» названия географических пунктов.
  - **6. При переработке** (если ДТ — документ об условиях переработки): норма выхода, наименование/количество/код продуктов переработки, способы идентификации, отходы, эквивалентная замена, операции переработки.
  - **7. При периодическом/временном декларировании**: `Поставка в период с ДД.ММ.ГГГГ по ДД.ММ.ГГГГ`.
  - **10. Контрольные (идентификационные) знаки**: количество + через «:» через «,» идентификационные номера (при маркировке по Решению СЕ ЕЭК № 70). Если маркировка после выпуска → `ПВ`.
  - **13. Маркировка средствами идентификации** (Честный ЗНАК, для РФ/РК/КР): общее количество кодов идентификации, кодовое обозначение уровня маркировки (`0` — единица товара, `1` — групповая упаковка, `2` — транспортная упаковка, `3` — набор, `4` — упаковка с разными кодами ТН ВЭД), коды идентификации, агрегированные коды. Подряд идущие → через «–». Не более 50 кодов на один уровень маркировки одного товара.
  - **15. Прослеживаемость**: количество товара в единице измерения прослеживаемости, условное обозначение и код единицы (по Классификатору единиц измерения ЕАЭС).
- **Источники**: наименование → техническое описание; упаковка → PL; контейнеры → транспортные документы.

### Графа 32 — Порядковый номер товара
- **Подраздел 1**: порядковый номер (от 1 до 999).
- **Подраздел 2** (при наличии):
  - `МПО` — международные почтовые отправления.
  - `ЭКГ` — экспресс-грузы.

### Графа 33 — Код товара (ТН ВЭД ЕАЭС)
- **Классификаторы**: ТН ВЭД ЕАЭС (подраздел 1), Классификатор дополнительной таможенной информации (подраздел 3).
- **Источники** (в порядке приоритета):
  1. Решение ФТС о классификации (если имеется).
  2. Техническое описание товара.
  3. Инвойс на товары — описание товара.
- **Три подраздела**:
  - **Подраздел 1**: 10-значный код ТН ВЭД ЕАЭС без пробелов.
  - **Подраздел 2**: буквенные обозначения (могут быть несколько, без пробелов):
    - `С` — товар свободен от запретов/ограничений (по коду подпадает, но по характеристикам — нет).
    - `И` — содержит ОИС из таможенного реестра. С 01.04.2025 — код по Классификатору сведений об ОИС.
    - `М` — включён в перечень маркируемых, но маркировка не распространяется.
    - `О` — товары разных наименований с одним кодом ТН ВЭД (РФ, КР).
    - `П` — товар подлежит прослеживаемости.
  - **Подраздел 3**: 4-значный код по Классификатору дополнительной таможенной информации (если применим).
- **AI-правило**: если есть решение ФТС о классификации → использовать его; иначе AI выдаёт top-N кандидатов + обоснование + confidence, помечает «нужна проверка».

### Графа 34 — Код страны происхождения
- **Классификатор**: Классификатор стран мира.
- **Два подраздела**:
  - **Подраздел «a»**: код страны происхождения данного товара (ISO alpha-2). Если происхождение неизвестно → `00`. Если маркировка указывает на ЕС без конкретной страны → `EU`.
  - **Подраздел «b»**: код страны для тарифных **преференций** (если в Графе 36 элемент 2 содержит код преференции). В иных случаях — не заполняется.
- **Источники** (в порядке приоритета):
  1. Сертификат происхождения.
  2. Декларация о соответствии — поле «страна изготовителя».
  3. Техническое описание — страна производителя.
- **Не заполняется**: при декларировании наличной валюты.

### Графа 35 — Вес брутто (кг)
- **Правило**: масса брутто (товар + вся упаковка, кроме контейнеров и транспортного оборудования) в килограммах.
- **Округление**:
  - > 1 кг: до 3 знаков после запятой.
  - < 1 г: до **6 знаков** после запятой.
- **Источники** (в порядке приоритета): PL, транспортные документы, сертификат взвешивания.
- **Расчёт**: суммировать вес брутто **только по строкам данного товара** в PL. При расхождении — приоритет PL.
- **Не заполняется**: при декларировании наличной валюты; товаров по ЛЭП или природного газа по трубопроводу.

### Графа 36 — Преференции
- **Классификатор**: Классификатор льгот по уплате таможенных платежей.
- **Структура — 4 элемента**:
  - **Элемент 1** — по таможенным сборам.
  - **Элемент 2** — по таможенной пошлине.
  - **Элемент 3** — по акцизу.
  - **Элемент 4** — по НДС.
- **Заполнение пустых элементов**:
  - Код из классификатора — если заявляется льгота/преференция.
  - `00` — если ставка существует, но льгота не заявляется.
  - `--` (прочерк) — для РФ, РБ, Армении, если обязанность по уплате данного платежа не возникает (нет ставки).
  - `Z` — для РК, КР (аналог прочерка).
- **Код `ПВ`** (в элементе 2) — если декларант планирует восстановить тарифные преференции после выпуска.
- **Источники** (в порядке приоритета):
  1. Сертификат происхождения (форма СТ‑1, Form A, EUR.1 и др.).
  2. Международные соглашения о преференциях (ЗСТ, СНГ и т.п.).
  3. Решения о предоставлении тарифных преференций.
- **AI-правило**: подсветить для проверки. AI может определить наличие сертификата происхождения, но финальные коды преференций требуют подтверждения.

### Графа 37 — Процедура
- **Классификаторы**: Классификатор видов таможенных процедур (подраздел 1), Классификатор особенностей перемещения товаров (подраздел 2).
- **Подраздел 1** — составной код `ААББ`:
  - `АА` — двузначный код заявляемой процедуры.
  - `ББ` — двузначный код предшествующей процедуры (кроме транзита), или `00` если нет.
  - Пример: `4000` (ИМ40, без предшествующей).
- **Подраздел 2**: трёхзначный код особенности перемещения. Если нет → `000`.
- **Источники**: контракт / договор купли-продажи, Графа 1 (тип процедуры).
- **Контроль**: поле согласовано с Графой 1. При несоответствии → `warning`.

### Графа 38 — Вес нетто (кг)
- **Правило**: масса нетто в килограммах.
  - Товар в первичной упаковке для розничной продажи (неотделима) → масса **с учётом** первичной упаковки.
  - В остальных случаях → масса **без** какой-либо упаковки.
  - Насыпом/навалом/наливом → общая масса товара.
- **Через «/»**: масса нетто без учёта **всех** видов упаковки (при наличии таких сведений в документах или на упаковке).
- **Округление**: аналогично Графе 35 (3 знака; < 1 г → 6 знаков).
- **Источники**: PL. Суммировать net weight по строкам **данного конкретного товара**.
- **Контроль**: нетто < брутто (гр. 35); если нетто > брутто → `warning`.
- **Не заполняется**: при декларировании наличной валюты; товаров по ЛЭП.

### Графа 39 — Квота
- **Источники**: лицензии / квотные свидетельства.
- **Правило**: указывается **остаток** выделенной квоты (без учёта декларируемых товаров) в единицах лицензии.

### Графа 40 — Общая декларация / предшествующий документ
- **Формат**: `номер_ДТ/порядковый_номер_товара`.
- **Правило**: регистрационный номер предшествующей ДТ (кроме транзита) + через «/» порядковый номер товара из Графы 32 предшествующей ДТ.
- При выпуске до подачи ДТ → регистрационный номер заявления о выпуске.

### Графа 41 — Дополнительные единицы
- **Классификатор**: Классификатор единиц измерения (ЕАЭС).
- **Правило**: количество товара в дополнительной единице измерения (если предусмотрена ТН ВЭД для данного кода). Далее через «/» условное обозначение и код единицы.
- **Точность**: до 2 знаков после запятой; если < 0.01 → до **6 знаков**.
- **Источники**: PL — единственный источник. Спецификация не используется.

### Графа 42 — Цена товара
- **Правило**: фактурная стоимость **конкретного товара** в валюте контракта.
- **Источники**: инвойс на товары (не транспортный инвойс) — для стоимости позиции; договор/контракт — для валюты.
- **Алгоритм**:
  1. Найти строку данного товара в инвойсе → взять `line_total` или `unit_price × quantity`.
  2. Валюта = из Графы 22 (источник — контракт).
  3. Если валюта в инвойсе ≠ валюта контракта → пересчёт + `warning`.
- **Безвозмездные поставки**: при безвозмездных поставках (в т.ч. в рамках возмездных договоров), а также при поставках без сделки → указать **«0»** (для РФ, РК, Армении). Для РБ и КР → указать оценочную стоимость.
- **Округление**: до 2 знаков.
- **Контроль**: `sum(items[].graph_42)` = подраздел 2 Графы 22.

### Графа 43 — Код МОС
- **Классификатор**: Классификатор методов определения таможенной стоимости (подраздел 1); Классификатор результатов таможенного контроля таможенной стоимости (подраздел 2 — заполняет таможня).
- **Подраздел 1**: код метода (1–6):
  - `1` — по стоимости сделки с ввозимыми товарами (основной).
  - `2` — с идентичными товарами, `3` — с однородными, `4` — вычитания, `5` — сложения, `6` — резервный.
- **Подраздел 2**: заполняется должностным лицом таможни (AI не трогает).
- **Не заполняется**: при декларировании валюты, ценных бумаг; при процедурах таможенного склада, уничтожения, отказа в пользу государства, специальной; в РБ (при вывозе).

### Графа 44 — Дополнительная информация / Представленные документы
- **Классификатор**: Классификатор видов документов и сведений.
- **Формат каждой строки**: `КОД/ПРИЗНАК НОМЕР ОТ ДАТА [ПО ДАТА_ОКОНЧАНИЯ]`
- **Признаки представления**:
  - `0` — документ НЕ представлен при подаче ДТ.
  - `1` — документ представлен при подаче ДТ.
  - `2` — не представлен (ранее прилагался к другой ДТ). После признака — номер, дата + регистрационный номер ДТ, к которой прилагался.
  - `3` — будет представлен после выпуска. После признака — запись об обязательстве: `ОБЯЗУЮСЬ ПРЕДСТАВИТЬ ДО ДД.ММ.ГГГГ`.
- **Перечень документов (по каждому товару)**:
  - Документы, подтверждающие запреты и ограничения (номер, дата, срок действия).
  - Транспортные документы (номера, даты).
  - Транзитные декларации.
  - Договор/контракт (номер, дата).
  - Коммерческие документы (инвойс, проформа и др.).
  - Решение о классификации (если есть).
  - Сертификат происхождения (номер, дата, код страны выдачи).
  - Декларация о происхождении (номер, дата).
  - Документы о льготах (наименование, номер, дата, срок, код вида платежа).
  - Документ об обеспечении (номер, дата).
  - Документ валютного контроля.
  - Документы об условиях переработки.
  - Документ об УЭО.
  - Документ о резидентстве СЭЗ/ОЭЗ.
- **Источники**: все загруженные пользователем документы (инвойс, транспортные документы, контракт, PL, сертификаты, доверенности, лицензии и т.д.).
- **AI-правило**: автоматически сформировать перечень из загруженных файлов; по каждому определить тип → код из классификатора, извлечь номер и дату.

### Графа 45 — Таможенная стоимость
- **Правило**: таможенная стоимость **конкретного товара** в рублях. Округление до 2 знаков (для Армении — до целого).
- **Метод 1** (основной — по стоимости сделки с ввозимыми товарами):
  1. Базовая цена = гр. 42 (стоимость данного товара из инвойса).
  2. + стоимость перевозки до границы ЕАЭС (из транспортного инвойса). Если общая сумма за партию → распределить по массе брутто: `freight_item = total_freight × (gross_weight_item / total_gross_weight)`.
  3. + страхование, комиссионные, упаковка, роялти (при наличии документов, пропорционально массе).
  4. Итог × курс ЦБ (гр. 23). Округлить до 2 знаков.
- **Источники**: инвойс на товары (гр. 42), транспортный инвойс (стоимость перевозки), документы по страхованию, документы по роялти/лицензиям, документы по упаковке, курс ЦБ (гр. 23).
- **Не заполняется**: при декларировании валюты, ценных бумаг; при процедурах таможенного склада, уничтожения, отказа в пользу государства, специальной.
- **Если есть ДТС-1** → использовать его. Иначе → рассчитать по методу 1 + пометка «предварительно — нет ДТС-1».

### Графа 46 — Статистическая стоимость
- **Формат**: в долларах США, без пробелов/разделителей, до 2 знаков. Пример: `12345.67`.
- **Источники**: гр. 45 (таможенная стоимость), гр. 42 (цена товара — запасной вариант), курс ЦБ USD на дату регистрации ДТ.
- **Расчёт** (для импорта):
  1. Основа — таможенная стоимость (гр. 45). Если гр. 45 не заполнена → цена (гр. 42).
  2. Привести к базису CIF (порт) / CIP (граница ЕАЭС): если место прибытия за пределами территории государства-члена назначения → добавить расходы на перевозку до границы этого государства.
  3. Пересчитать в USD по курсу ЦБ на дату регистрации ДТ: `стоимость_в_рублях / курс_USD`.
- **Расчёт** (для экспорта): привести к базису FOB (порт) / DAP (граница государства-члена отправления).
- **Округлить** до 2 знаков.

## 7) Платежи (графы 47–48, B)

### Графа 47 — Исчисление платежей
- **Классификаторы**: Классификатор видов налогов, сборов и иных платежей; Классификатор особенностей уплаты таможенных и иных платежей.
- **Колонки**: Вид | Основа начисления | Ставка | Сумма | СП.
- **Колонка «Вид»**: код вида платежа. Основные: `1010` — сборы за таможенные операции, `2010` — ввозная пошлина, `5010` — НДС, `4010` — акциз.
- **Колонка «Основа начисления»**: база исчисления (таможенная стоимость, масса и т.д.). Округление: для РФ — до 2 знаков (в деньгах); до 6 знаков (в весе).
- **Колонка «Ставка»**: размер ставки.
- **Колонка «Сумма»**: исчисленная сумма. Для РФ — до 2 знаков.
- **Колонка «СП»** (специфика платежа):
  - `ИУ` — платёж уплачен в полном размере.
  - `ЗН` — НДС уплачен методом зачёта (РК).
  - `ОП` — отсрочка всей суммы.
  - `ОЧ` — отсрочка части суммы.
  - `РП` — рассрочка.
  - `УР` — платёж не подлежит уплате (условно начислен).
  - `УН` — платёж начислен без учёта льгот (для дополнительной строки).
  - `УМ` — платёж освобождён от уплаты (в дополнительной строке).
  - `ВВ` — частичная уплата при временном ввозе (3% в месяц).
- **Комбинированная ставка**: исчисляется в **две строки** (адвалорная + специфическая). В Графе B — одной строкой.
- **Не заполняется**: при декларировании валюты и ценных бумаг; при процедурах таможенного склада, реэкспорта, специальной (пошлины/налоги).
- **AI-правило**: считать через сервис расчёта по ставкам ТН ВЭД, не угадывать.

### Графа B — Подробности подсчёта
- **Классификаторы**: Классификатор видов налогов, сборов и иных платежей; Классификатор валют; Классификатор способов уплаты таможенных и иных платежей.
- **Содержание**: суммы **уплаченных** платежей за все товары ДТ, в разрезе документов подтверждения.
- **Формат строки** (элементы через «–», без пробелов):
  - **Элемент 1**: код вида платежа (по Классификатору).
  - **Элемент 2**: сумма уплаченного платежа.
  - **Элемент 3**: цифровой код валюты уплаты (по Классификатору валют). Не указывается для РК.
  - **Элемент 4**: номер документа подтверждения уплаты. Не указывается для РК и РФ.
  - **Элемент 5**: для РФ — не указывается; для РБ — дата уплаты.
  - **Элемент 6**: способ уплаты (по Классификатору). Не указывается для РК и РФ.
  - **Элемент 7**: для РФ и РБ — налоговый номер лица, уплатившего платёж (ИНН для РФ, УНП для РБ).
- **Округление суммы**: для РФ — до 2 знаков; для РК — до целого.
- **При централизованном порядке уплаты**: элементы 4–6 не заполняются.

### Графа 48 — Отсрочка платежей
- **Классификатор**: Классификатор видов налогов, сборов и иных платежей.
- **Формат**: `КОД_ПЛАТЕЖА–ДАТА_ПОСЛЕДНЕГО_ДНЯ_УПЛАТЫ` (дд.мм.гггг).
  - При рассрочке — даты каждого этапа уплаты.
- **Не заполняется**: если отсрочка/рассрочка не предоставлялась.
