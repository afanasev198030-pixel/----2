
# Правила AI‑заполнения ДТ по графам (v4.0)

Источник требований: официальная инструкция по заполнению таможенной декларации (ЕАЭС) + ТЗ v3.0.

Цель этого документа — описать **правила**, по которым AI заполняет ДТ (декларацию на товары) **на основе документов**, с контролем источников, проверками и флагами "нужна ручная проверка".

## 1) Общие принципы (как AI должен работать)

- **Только evidence‑based**: AI заполняет графу, только если может сослаться на документ‑источник (контракт/инвойс/PL/транспорт/сертификаты). Если данных нет — оставляет пусто и ставит "нужно уточнить".
- **Приоритет источников (если конфликтуют)**:
  1) **Подтверждающие/разрешительные** документы (сертификат происхождения, решения ФТС, лицензии/квоты, ДТС‑1 и т.п.)
  2) **Контракт + доп. соглашения**
  3) **Инвойс**
  4) **Packing List / спецификация**
  5) **Транспортные документы**
  6) **Внутренние справочники/профиль компании** (ИНН/КПП, реквизиты)
  7) **Модельные догадки** (разрешены только как "кандидаты", не как финальное значение для критичных граф)
- **Нормализация**:
  - страны: ISO 3166‑1 alpha‑2 (CN, RU, DE…)
  - валюта: ISO 4217 (USD, EUR, CNY…)
  - ТН ВЭД: строго 10 цифр (01–97 в первых двух)
  - числа: разделители пробел/запятая → привести к float/Decimal
- **Валидации и "красные флаги"**: если value не проходит формат/диапазон/связь между графами — AI не "прибивает гвоздями", а просит подтверждение.
- **Сопровождающая мета‑информация (рекомендовано)**: для каждого значения хранить:
  - `value`
  - `sources[]` (какой документ/страница/фрагмент)
  - `confidence` (0..1)
  - `notes` (почему выбран именно этот источник / что конфликтует)

## 2) Документы (минимальный набор и классификация)

По ТЗ обязательные документы для импорта/экспорта:
- Контракт/договор (+ доп. соглашения)
- Инвойс на товары
- Упаковочные листы
- Спецификация (общая на все товары из заказа)
- Транспортные документы (CMR / B/L / AWB / CIM)
- Транспортный инвойс
- Сертификат происхождения (СТ‑1 / Form A / EUR.1), если необходимо
- Application (заявка)

AI должен:
- сначала **классифицировать** каждый файл по типу (invoice/contract/packing/transport/origin_certificate/other),
- затем извлечь факты в единый "паспорт сделки",
- затем заполнить графы по правилам ниже.

## 3) Правила заполнения граф (шапка ДТ)

### Графа 1 — Декларация (тип/процедура)
- **Официальное описание**: Тип декларации: ИМ (импорт), ЭК (экспорт), ТТ (транзит) и др. Три подраздела: таможенная процедура, тип ДТ (ЭД — электронная), страна-член (RU).
- **Источники**: пользователь выбирает тип при создании декларации.
- **Правило**: заполнять код процедуры в соответствии с выбранным типом. Пример: `ИМ40` — выпуск для внутреннего потребления.
- **Если не найдено**: предложить ИМ40 для импорта с флагом "проверьте".

### Графа 2 — Отправитель/Экспортёр
- **Официальное описание**: Сведения об иностранном отправителе товара: наименование, адрес, страна. При наличии нескольких отправителей — делается отметка «Различные».
- **Источники**: контракт ("Продавец"), инвойс (Seller), рег. документы иностранной компании.
- **Заполнить**: наименование, адрес, страна (ISO 3166-1 alpha-2).
- **Валидация**: страна = не RU (обычно для импорта).

### Графа 3 — Формуляры (листы ДТ)
- **Официальное описание**: Указывает порядковый номер текущего листа и общее количество листов ДТ.
  - Подраздел 1: порядковый номер текущего листа (X).
  - Подраздел 2: общее количество листов ДТ, включая основной и все добавочные (Y).
  - Формат: `X/Y`, например `2/3`. На основном листе не заполняется.
- **Источники**: инвойс на товары (количество позиций — только отсюда).
- **Правило расчёта**:
  - Основной лист вмещает 1 товар; каждый добавочный лист — до 3 товаров.
  - `total_sheets = 1 + ceil((total_items - 1) / 3)` при `total_items > 1`; при 1 товаре — только основной лист, Графа 3 не заполняется.
  - На основном листе (лист 1) Графа 3 не заполняется.
  - На каждом добавочном листе: X = номер листа (начиная с 2), Y = total_sheets.

### Графа 4 — Отгрузочные спецификации
- **Официальное описание**: Количество приложенных к ДТ отгрузочных спецификаций (упаковочных листов).
- **Источники**: отгрузочные спецификации к контракту.
- **Правило**: заполняется **только если есть отгрузочные спецификации**. В остальных случаях оставить пустым.

### Графа 5 — Всего товаров
- **Официальное описание**: Общее количество наименований товаров, указанных в ДТ (по всем листам).
- **Правило**: `total_items_count` = количество товарных **наименований (позиций)**, а не штук.
- **Источники**: инвойс на товары + Packing List (PL).
  - **⚠️ Важно**: использовать именно **инвойс на товары**, а не транспортный инвойс (инвойс за перевозку).
  - В инвойсе: считать количество строк товарных позиций (line items).
  - В PL: считать количество **уникальных наименований товаров**, а не общее количество штук/мест.
- **Контроль**: если есть и инвойс, и PL — количество наименований должно совпадать. При расхождении — поднять флаг и показать обе цифры.

### Графа 6 — Количество мест
- **Официальное описание**: Общее количество грузовых мест по всем товарам в ДТ.
- **Источники**: packing list (итог мест), транспортные накладные.
- **Правило**: брать total packages из PL; если конфликт с транспортным документом — вывести обе цифры и флаг.

### Графа 7 — Особенности декларирования
- **Официальное описание**: Код особенности декларирования товаров из классификатора особенностей таможенного декларирования.
- **Правило**: заполняется только если для данной партии есть применимый код в классификаторе. Если особенностей нет или код не предусмотрен — оставить пустым.
- **AI-правило**: не заполнять автоматически. Подсветить графу для пользователя с пояснением: «Укажите код особенности декларирования из классификатора или оставьте пустым».

### Графа 8 — Получатель
- **Официальное описание**: Сведения о получателе товара на территории ЕАЭС: наименование, ИНН/КПП, адрес.
- **Заполнить**: наименование, ИНН, КПП, ОГРН, адрес.
- **Источники** (в порядке приоритета):
  1. Профиль компании в системе — наивысший приоритет.
  2. Контракт — раздел «Buyer» / «Покупатель».
  3. Инвойс на товары — поле «Buyer» / «Покупатель».
- **Приоритет языка**: данные на **русском языке** приоритетнее, чем на английском. Если в контракте есть и русская, и английская версия реквизитов — брать русскую.

### Графа 9 — Лицо, ответственное за финансовое урегулирование
- **Официальное описание**: Лицо, несущее финансовую ответственность за сделку. Заполняется, если отличается от получателя (гр. 8).
- **Источники**: доверенность, договор с таможенным представителем (брокером).
- **Правило**:
  - Если есть доверенность или договор с брокером — извлечь реквизиты из документа.
  - Если таких документов нет — заполнить фиксированным текстом: **«СМ. ГРАФУ 14 ДТ»**.

## 4) Правила по странам (графы 10–17)

### Графа 10 — Станция 1-го назначения
- **Правило**: не заполняется.

### Графа 11 — Торгующая страна
- **Официальное описание**: Страна местонахождения иностранного контрагента (продавца).
- **Формат**: ISO 3166-1 alpha-2.
- **Источники** (в порядке приоритета):
  1. Контракт — страна из реквизитов продавца/поставщика.
  2. Сертификат происхождения — страна контрагента.
  3. Инвойс на товары — страна из реквизитов продавца.
- **Если конфликт**: приоритет контракт > сертификат происхождения > инвойс.

### Графа 12 — Общая таможенная стоимость
- **Официальное описание**: Суммарная таможенная стоимость всех товаров по ДТ в валюте государства — члена ЕАЭС (в рублях для РФ).
- **Правило**: сумма из граф 45 основного и добавочных листов. Округление до 2 знаков. Требует курс ЦБ (гр. 23).

### Графа 13 — Резервная
- **Официальное описание**: Не заполняется участниками ВЭД. Предназначена для служебных отметок таможенного органа.
- **AI-правило**: пропустить, не заполнять.

### Графа 14 — Декларант
- **Официальное описание**: Сведения о лице, подающем ДТ: краткое наименование организации, место нахождения, ИНН/КПП, ОГРН.
- **Правило**:
  - Если есть доверенность или договор с брокером — заполнить данными брокера/таможенного представителя.
  - Если таких документов нет — заполнить данными **Получателя** (как в Графе 8).
- **Источники** (в порядке приоритета):
  1. Профиль компании в системе.
  2. Контракт — раздел «Buyer» / «Покупатель».
  3. Инвойс на товары — поле «Buyer» / «Покупатель».
- **Приоритет языка**: данные на **русском языке** приоритетнее английских.
- **Заполнить**: краткое наименование, место нахождения, ИНН, КПП, ОГРН.

### Графа 15 / 15А — Страна отправления
- **Официальное описание**: Страна, из которой непосредственно прибыл товар (не страна происхождения). Определяется на основании сведений в транспортных (перевозочных) документах, по которым осуществляется международная перевозка.
  - Графа 15 — словесное наименование страны.
  - Графа 15А — буквенный код страны (ISO 3166-1 alpha-2).
- **Источники** (в порядке приоритета):
  1. Транспортные документы AWB / CMR — поле «Departure» / «From».
  2. Транспортный инвойс — поле «Departure».

### Графа 16 — Страна происхождения
- **Официальное описание**: Страна, в которой товар был произведён или подвергнут достаточной обработке. Используется для определения ставки пошлины и преференций.
- **Источники**: сертификат происхождения, декларация соответствия (поле «страна изготовителя»).
- **Правила заполнения**:
  - Если документы/маркировка указывают на **Европейский союз** (без конкретной страны) → записать: **«ЕВРОСОЮЗ»**.
  - Если в ДТ несколько товаров из **разных стран** ИЛИ страна хотя бы одного товара неизвестна → записать: **«РАЗНЫЕ»**.
  - Если страна происхождения **всех** товаров неизвестна → оставить пустым и **подсветить для пользователя** (уровень `warning`): «Укажите страну происхождения товаров».
  - Не заполняется при декларировании наличной валюты (банкноты, монеты), перемещаемой в связи с реализацией товаров на бортах воздушных судов и иных видах транспорта.

### Графа 17 / 17А — Страна назначения
- **Официальное описание**: Краткое название страны, известной на день подачи ДТ в качестве страны назначения, где товары будут потребляться, использоваться или подвергнуты дальнейшей переработке. Если такая страна неизвестна — указывается страна доставки товаров.
- **Источники**: транспортные документы (поле «страна прибытия» / destination).
- **Правила заполнения**:
  - В общем случае — страна назначения из транспортных документов.
  - Если конечная страна потребления неизвестна — указать известную страну доставки.
  - Значение **«НЕИЗВЕСТНА»** указывается только в трёх случаях:
    1. Декларирование вывозимых товаров под процедуру беспошлинной торговли.
    2. Подача неполной или периодической ДТ (сведения будут предоставлены дополнительно).
    3. Подача временной (дополнительной временной) ДТ (сведения будут заявлены в полной ДТ).
  - В остальных случаях «НЕИЗВЕСТНА» не используется.
- **Графа 17А**: двузначный буквенный код страны из Графы 17 в соответствии с классификатором стран мира (ISO 3166-1 alpha-2). Заполняется автоматически на основе значения Графы 17.
- **Контроль**: Графа 17 обычно совпадает с Графой 10. При расхождении — поднять флаг для проверки пользователем.

## 5) Правила по транспорту (графы 18–30)

### Графа 18 — Идентификация и страна регистрации ТС при отправлении
- **Официальное описание**: Идентификационный номер ТС при прибытии/убытии. Подраздел 1: кол-во ТС + номера через «;». Подраздел 2: код страны регистрации. При ж/д — подраздел 2 не заполняется. Не заполняется при отсутствии международной перевозки и для МПО.
- **Формат по типу транспорта**:
  - Авто: регистрационные номера (активное ТС + прицепы через «/»)
  - ЖД: номера вагонов
  - Море: наименования судов
  - Авиа: номера рейсов
  - Трубопровод/ЛЭП: способ транспортировки

### Графа 19 — Контейнер
- **Официальное описание**: Признак перевозки в контейнере: «1» — да, «0» — нет.
- **Источники**: транспортные документы AWB / CMR / B/L — наличие номера контейнера или явного указания на контейнерную перевозку.
- **Правило**: если в транспортном документе присутствует номер контейнера или указано «container» — ставить «1»; в остальных случаях — «0».

### Графа 20 — Условия поставки
- **Официальное описание**: Код поставки по Инкотермс и место поставки — наименование географического пункта, к которому относится базис.
- **Источники**: договор поставки (основной и единственный источник).
- **Заполнить**: код Инкотермс (3 буквы) + наименование географического пункта. Например: `CIF Москва`, `FOB Shanghai`.

### Графа 21 — Идентификация и страна регистрации активного ТС на границе
- **Официальное описание**: ТС, пересекающее таможенную границу (может отличаться от гр. 18 при перегрузке).
- **Источники**: транспортные документы AWB / CMR / B/L.
- **Формат**: `[кол-во ТС]: [идентификатор1];[идентификатор2]` — без пробелов вокруг «;».
- **Подраздел 1** — количество ТС и идентификаторы по типу транспорта:
  - **Авто**: регистрационные номера транспортных средств.
  - **Море / река**: названия судов.
  - **Авиа**: номера рейсов.
  - **Трубопровод / ЛЭП**: способ транспортировки — «газопровод», «нефтепровод», «нефтепродуктопровод», «линии электропередачи».
- **Подраздел 2** — код страны регистрации ТС (ISO 3166-1 alpha-2):
  - Если состав ТС — код страны регистрации **тягача** (приводящего в движение).
  - Если страна неизвестна → **«00»**.
  - Если несколько ТС из разных стран → **«99»**.
  - При трубопроводном транспорте и ЛЭП — **не заполняется**.

### Графа 22 — Валюта и общая фактурная стоимость
- **Официальное описание**: Код валюты и общая стоимость товаров.
  - **Подраздел 1**: код валюты — берётся **только из договора/контракта купли-продажи** (ISO 4217). Инвойс не является источником для валюты.
  - **Подраздел 2**: общая стоимость = сумма значений Графы 42 по всем позициям основного и добавочных листов ДТ.
- **Источники**:
  - Код валюты → **договор/контракт купли-продажи** (единственный источник).
  - Сумма → рассчитывается как `sum(items[].graph_42)`. Для сравнения использовать инвойс на товары (итоговая сумма).
- **Как найти валюту в контракте**: искать ключевые фразы:
  - `«Валютой Контракта являются...»`
  - `«Валюта контракта —...»`
  - `«Contract currency is...»`
  - `«Payment currency:...»`
  - `«расчёты производятся в...»`
  После ключевой фразы следует название или код валюты (доллары США → USD, евро → EUR, юани → CNY и т.д.). Привести к формату ISO 4217.
- **Контроль**: рассчитанная сумма должна совпадать с итогом инвойса на товары. При расхождении — поднять флаг.

### Графа 23 — Курс валюты
- **Официальное описание**: Курс иностранной валюты к валюте государства — члена ЕАЭС на дату регистрации ДТ.
- **Правило**: использовать курс ЦБ РФ **на дату принятия ДТ**.

### Графа 24 — Характер сделки
- **Официальное описание**: Код характера внешнеторговой сделки по классификатору.
- **Источники**: контракт / договор.
- **Коды**: 01 купля‑продажа, 02 бартер, 03 безвозмездная.
- **Если не найдено**: дефолт 01 + флаг "проверьте".

### Графа 25 — Вид транспорта на границе
- **Официальное описание**: Код вида транспорта, которым товар пересекает таможенную границу.
- **Источники**: транспортные документы AWB / CMR / B/L.
- **Коды**: 10 море, 20 ж/д, 30 авто, 40 воздух.
- **AI-правило**: определить тип транспорта по виду документа: AWB → 40 (воздух), CMR → 30 (авто), B/L → 10 (море).

### Графа 26 — Вид транспорта внутри страны
- **Правило**: не заполняется.

### Графа 27 — Место погрузки/разгрузки
- **Правило**: не заполняется.

### Графа 28 — Финансовые и банковские сведения
- **Правило**: не заполняется.

### Графа 29 — Таможня на границе
- **Официальное описание**: Код таможенного органа места ввоза/вывоза. Для РФ: 8-значный код. Через РБ/КЗ: «ZZZ XXXXX». При нескольких пунктах — все коды.
- **AI‑правило**: определить место прибытия из транспортных документов и сопоставить с классификатором таможенных органов.

### Графа 30 — Местонахождение товаров
- **Официальное описание**: Место нахождения товара на момент подачи ДТ: СВХ, склад декларанта, порт и т.д. Код и наименование места хранения.
- **Источники**: транспортные документы AWB / CMR / B/L (место доставки / place of delivery).
- **Правило**: определить место нахождения товара из транспортных документов. Если информация не найдена — оставить пустым и подсветить поле пользователю (уровень `warning`): «Укажите место нахождения товаров».

## 6) Правила по товарным позициям (графы 31–46, для каждой позиции)

> ### ⚠️ Ключевое правило раздела: данные — только по конкретному товару
>
> Графы 31–46 заполняются **отдельно для каждой товарной позиции** (каждого наименования товара в декларации). Это означает:
> - **Веса (гр. 35, 38)** — только масса данного конкретного товара, не суммарная масса всей партии.
> - **Цена (гр. 42)** — только стоимость данной позиции, не итог по всему инвойсу.
> - **Таможенная стоимость (гр. 45)** — рассчитывается по данной позиции, а не по всем товарам вместе.
> - **Статистическая стоимость (гр. 46)** — рассчитывается по данной позиции.
> - **Количество и упаковка (гр. 31, 41)** — только по данному наименованию в PL, не суммарно.
>
> **Как находить данные по конкретному товару**: искать в документах (PL, инвойс) строки/записи, относящиеся именно к данному наименованию товара (по названию, описанию, артикулу), и суммировать только их, если товар занимает несколько строк.

> **Важно о спецификации**: Спецификация является **вспомогательным** документом. Она содержит полный список товаров заказа, но на таможню приезжают только товары из **инвойса на товары** и **Packing List**. Поэтому:
> - **Список товарных позиций** → берётся из **инвойса на товары** (приоритет) или Packing List.
> - **Количество и веса** → берутся из **Packing List** (приоритет) или инвойса.
> - Спецификация используется только для **перекрёстной сверки** (контроль расхождений), но **не** как источник данных для заполнения граф.

### Графа 31 — Грузовые места и описание товаров
- **Официальное описание**: Подробное описание товара с указанием сведений о грузовых местах и упаковке. Заполняется на каждый товар отдельно. Сведения указываются с новой строки с проставлением порядкового номера.
- **Структура заполнения**:
  - **1. Наименование товара** — брать из документа «Техническое описание». Описание должно быть полным и точным, без обобщений «Item 1».
  - **2. Грузовые места и упаковка** — брать из Packing List (PL):
    - Общее количество грузовых мест, занятых товаром.
    - Если товар занимает места не полностью — дополнительно в скобках: `(X часть места)`.
    - Коды видов упаковки по Классификатору видов груза, упаковки и упаковочных материалов, через «-» количество упаковок каждого вида.
    - Пример: `3, 21-2, 4G-1` (3 места, 2 упаковки вида 21, 1 упаковка вида 4G).
- **Источники**:
  - Наименование → документ «Техническое описание».
  - Грузовые места и тип упаковки → Packing List.

### Графа 32 — Порядковый номер товара
- **Официальное описание**: Порядковый номер декларируемого товара (от 1 до 999). На основном листе — не более 1 товара, на добавочном — до 3.
- **Правило**: присваивать автоматически по порядку позиций.

### Графа 33 — Код товара (ТН ВЭД ЕАЭС)
- **Официальное описание**: 10-значный код по ТН ВЭД ЕАЭС. Определяет ставки пошлин, налогов, запреты и ограничения.
- **Критично**: должен быть подтверждён документально.
- **AI‑правило**: если есть решение ФТС — использовать его; иначе AI выдаёт top‑N кандидатов с обоснованием и confidence, помечает как «нужна проверка».

### Графа 34 — Код страны происхождения
- **Официальное описание**: Двузначный буквенный код страны происхождения товара (CN, DE, US) по классификатору.
- **Формат**: ISO 3166‑1 alpha‑2.
- **Источники** (в порядке приоритета):
  1. Сертификат происхождения.
  2. Декларация о соответствии — поле «страна изготовителя».
  3. Техническое описание — страна производителя.

### Графа 35 — Вес брутто (кг)
- **Официальное описание**: Масса брутто (с упаковкой) в килограммах. Округляется до 3 знаков после запятой.
- **Источники** (в порядке приоритета): Packing List (PL), транспортные документы, сертификат взвешивания.
- **Расчёт**: суммировать вес брутто **только по строкам данного конкретного товара** в PL (не общий вес всей партии). При расхождении между источниками — приоритет PL.

### Графа 36 — Преференции
- **Официальное описание**: Отражает, заявляются ли тарифные преференции и/или льготы по пошлине, акцизу, НДС, сборам по каждому товару. Коды из Классификатора льгот/преференций (национальный + ЕАЭС).
- **Источники** (в порядке приоритета):
  1. Сертификат происхождения (форма СТ‑1, Form A, EUR.1 и др.).
  2. Международные соглашения о преференциях (ЗСТ, СНГ и т.п.).
  3. Решения о предоставлении тарифных преференций.
- **Структура — 4 элемента** (по Инструкции к Решению КТС № 257):
  - **Элемент 1** — тарифная преференция по ввозной/вывозной пошлине (ЗСТ, СНГ, наименее развитые страны и др.).
  - **Элемент 2** — льготы по таможенной пошлине (освобождение, пониженная ставка).
  - **Элемент 3** — льготы по акцизу.
  - **Элемент 4** — льготы по НДС (при ввозе).
- **Заполнение пустых элементов**:
  - `00` — если ставка существует, но льгота/преференция не заявляется.
  - `--` — если обязанность по уплате данного платежа вообще не возникает (нет ставки, товар не облагается).
- **Важно**: применимые преференции и льготы определяются в соответствии с кодом ТН ВЭД товара (Графа 33). Разные коды ТН ВЭД могут иметь разные ставки и льготы.
- **AI-правило**: подсветить пользователю для проверки — AI может определить наличие сертификата происхождения, но финальные коды преференций требуют подтверждения.

### Графа 37 — Процедура
- **Официальное описание**: Содержит 2 подраздела.
- **Подраздел 1** — составной код по схеме `ААББ`:
  - `АА` — двузначный код заявляемой таможенной процедуры (по классификатору видов таможенных процедур).
  - `ББ` — двузначный код предшествующей таможенной процедуры (если товар ранее помещался под иную процедуру, кроме транзита), либо код процедуры, под которую помещались товары, из которых изготовлен декларируемый товар. Если предшествующей процедуры не было — `00`.
  - Пример: `4000` (процедура ИМ40, без предшествующей).
- **Подраздел 2** — трёхзначный код особенности перемещения товаров (по классификатору особенностей перемещения). Если особенностей нет — `000`.
- **Источники**: контракт / договор купли-продажи.
- **Правило**: поле согласовано с Графой 1. При несоответствии — подсветить пользователю (`warning`).

### Графа 38 — Вес нетто (кг)
- **Официальное описание**: Масса нетто (без упаковки) в килограммах.
- **Источники**: Packing List (PL).
- **Правило**: суммировать net weight **только по строкам данного конкретного товара** в PL — не брать значение одной произвольной строки и не брать суммарный нетто всей партии.
- **Контроль**: нетто < брутто (гр. 35); если нетто > брутто — поднять warning.

### Графа 39 — Квота
- **Официальное описание**: Сведения о тарифной квоте, если товар ввозится в рамках квоты.
- **Источники**: лицензии/квотные свидетельства/разрешения.

### Графа 40 — Общая декларация/предшествующий документ
- **Официальное описание**: Регистрационные номера предшествующих таможенных документов: транзитных деклараций, ДТ по предшествующим процедурам.
- **Формат**: номер предыдущей ДТ + дата регистрации.

### Графа 41 — Единица дополнительных единиц
- **Официальное описание**: Количество товара в дополнительной единице измерения (штуки, литры, пары и т.д.). Заполняется только если дополнительная единица предусмотрена для кода ТН ВЭД.
- **Источники**: Packing List (PL) — единственный источник. Спецификация не используется.
- **AI‑правило**: проверить по коду ТН ВЭД (гр. 33), предусмотрена ли дополнительная единица измерения. Если предусмотрена — взять количество из PL по конкретной позиции товара.

### Графа 42 — Цена товара
- **Официальное описание**: Фактурная стоимость **конкретного товара** в валюте контракта. База для расчёта таможенной стоимости и НДС.
- **Источники**: инвойс на товары (не транспортный инвойс) — для стоимости позиции; договор/контракт купли-продажи — для определения валюты.
- **Алгоритм**:
  1. Найти строку данного товара в инвойсе на товары — взять `line_total` или `unit_price × quantity` именно для этой позиции (не итог инвойса).
  2. Определить валюту контракта из графы 22 (источник — договор/контракт).
  3. Если валюта в инвойсе совпадает с валютой контракта — использовать напрямую.
  4. Если валюта в инвойсе **отличается** от валюты контракта — пересчитать по курсу ЦБ на дату инвойса и подсветить пользователю для проверки.
- **Контроль**: сумма гр. 42 по всем позициям = подраздел 2 гр. 22.

### Графа 43 — Код МОС
- **Официальное описание**: Код метода определения таможенной стоимости:
  - 1 — по стоимости сделки с ввозимыми товарами
  - 2 — по стоимости сделки с идентичными товарами
  - 3 — по стоимости сделки с однородными товарами
  - 4 — метод вычитания
  - 5 — метод сложения
  - 6 — резервный метод
  - Второй подраздел — код признака корректировки таможенной стоимости (КТС).
- **Если нет данных**: дефолт 1 + флаг "проверьте".

### Графа 44 — Дополнительная информация / Представленные документы
- **Официальное описание**: Перечень всех документов, на основании которых заполнена ДТ и подтверждаются заявленные сведения. Каждый документ указывается с кодом вида документа по классификатору ФТС, номером и датой.
- **Источники**: все загруженные пользователем документы (инвойс, транспортные документы, контракт, PL, сертификаты, доверенности, лицензии и т.д.).
- **AI-правило**: автоматически сформировать перечень из всех загруженных файлов. По каждому файлу определить тип документа, извлечь его номер и дату, подобрать соответствующий код по классификатору ФТС. Вывести список в виде: `КОД; НОМЕР; ДАТА`.
- **Формат строки**: `КОД_ФТС; №НОМЕР; ДД.ММ.ГГГГ`.

### Графа 45 — Таможенная стоимость
- **Официальное описание**: Таможенная стоимость **конкретного товара** в рублях. Округляется до 2 знаков. Не заполняется при декларировании товаров различных наименований с одним кодом ТН ВЭД и при декларировании национальной валюты.
- **Важно**: рассчитывается по данной позиции, не суммарная таможенная стоимость всей декларации.
- **Метод 1 — по стоимости сделки с ввозимыми товарами** (основной):
  1. **Базовая цена** = стоимость **данного товара** из инвойса на товары (гр. 42), не итог инвойса.
  2. **Прибавить стоимость перевозки** до границы ЕАЭС из транспортного инвойса:
     - Если транспортный инвойс содержит **общую стоимость за всю партию** — распределить по **массе брутто**:
       ```
       freight_item = total_freight × (gross_weight_item / total_gross_weight_all_items)
       ```
     - Если транспортный инвойс содержит стоимость **по данному товару отдельно** — использовать напрямую.
  3. **Прибавить** (только если загружены соответствующие документы):
     - Страхование груза (пропорционально массе брутто данного товара).
     - Комиссионные и вознаграждения посредникам (кроме комиссионера по покупке).
     - Стоимость упаковки и упаковочных работ.
     - Роялти и лицензионные платежи (если связаны с данным товаром и являются условием продажи).
  4. **Итоговую сумму** перевести в рубли по курсу ЦБ (гр. 23). Округлить до 2 знаков.
- **Источники**: инвойс на товары (гр. 42), транспортный инвойс, документы по страхованию, документы по роялти/лицензиям, документы по упаковке.
- **Правило**: если есть ДТС‑1 — использовать его. Иначе рассчитать по методу 1 и пометить «предварительно — нет ДТС-1».

### Графа 46 — Статистическая стоимость
- **Официальное описание**: Статистическая стоимость **по данной товарной подсубпозиции** в долларах США. Вносится без пробелов, разделителей и знака валюты, с точностью до двух знаков после запятой (например: `12345.67`).
- **Важно**: рассчитывается по конкретному товару (не суммарно по всей декларации).
- **Алгоритм расчёта**:
  1. **Определить основу** (по данному товару):
     - Основной источник — таможенная стоимость данного товара из гр. 45.
     - Если гр. 45 не заполнена — использовать цену данного товара из гр. 42.
     - Если не заполнены ни 42, ни 45 — использовать фактически уплаченную цену по коммерческим документам для данной позиции.
  2. **Привести к нужному базису**:
     - Импорт: обычно совпадает с гр. 45 (уже на базисе CIF/граница ЕАЭС).
     - Экспорт: привести к условиям FOB последний порт убытия или DAP/граница государства-члена ЕАЭС (добавить/исключить транспортные расходы до/после границы).
  3. **Пересчитать в доллары США**: использовать официальный курс ЦБ на дату регистрации ДТ.
  4. **Округлить** по математическим правилам до 2 знаков.
- **Источники**: гр. 45 (таможенная стоимость), гр. 42 (цена товара как запасной вариант), курс ЦБ USD на дату регистрации.

## 7) Платежи (графы 47–49)

### Графа 47 — Исчисление платежей
- **Официальное описание**: Таблица таможенных платежей. Колонки: Вид (2010 — пошлина, 5010 — НДС, 1010 — сборы), Основа начисления, Ставка, Сумма, СП (ИУ — уплачен, УР — условно, УМ — освобождён, ОП — отсрочка).
- **AI‑правило**: считать через сервис расчёта по ставкам ТН ВЭД, не угадывать.

### Графа 48 — Отсрочка платежей
- **Официальное описание**: Код платежа + номер и дата НПА + дата последнего дня уплаты. Разделитель «–» без пробелов. Не заполняется, если отсрочка не предоставлялась.

### Графа 49 — Наименование склада (СВХ)
- **Источники**: договор хранения, реестр СВХ, разрешение.
- **Заполнить**: наименование, адрес, номер в реестре.

## 8) Завершение (графа 54)

- **Официальное описание**: Сведения о лице, составившем ДТ. №1 — данные таможенного представителя (номер свидетельства, дата и номер договора с декларантом, ИНН/КПП). Не заполняется при самостоятельном декларировании. №2 — ФИО.
- **Формат**: `[Город], [ДД.ММ.ГГГГ]`.

## 9) Рекомендованный формат результата AI (для интеграции)

Результат AI должен включать:
- "плоские" значения для `Declaration` и массива `items[]`,
- `evidence_map`: по каждой графе → источники и confidence,
- `issues[]`: список конфликтов/пропусков, которые блокируют подачу.
