# Roadmap: Enterprise Customs Automation Platform

Этот план описывает эволюцию текущего проекта от "умного парсера" до полноценной экосистемы ВЭД (Внешнеэкономической деятельности), включающей логистику, комплаенс, финансы и автоматизированный клиентский сервис.

## Фаза 1: Умный Intake и AI-Стратегии (Ближайший приоритет)
*Цель: Избавить клиента от необходимости заходить в веб-интерфейс для старта работы и сделать логику парсинга гибкой.*

1. **Email-Intake Шлюз (Email-to-Declaration)** - *В ОЧЕРЕДИ*
   - Создание сервиса, который слушает выделенный почтовый ящик (IMAP) или принимает вебхуки от почтового сервера.
   - Автоматическое извлечение вложений (PDF, Excel) из писем клиентов.
   - Отправка файлов в существующий `POST /api/v1/ai/parse-smart`.
   - Автоматическое создание черновика декларации в БД с привязкой к клиенту (по email отправителя).
   - *Технологии:* Python `imaplib` / `aiosmtpd`, Celery для фоновых задач.

2. **AI-Стратегии (Global Rules Engine)** - *В ПРОЦЕССЕ (Backend ГОТОВ, Frontend нужен)*
   - **(СДЕЛАНО)** Создан мощный движок правил `rules_engine.py` и справочник `declaration_mapping_v3.yaml`.
   - **(СДЕЛАНО)** Созданы модели `GraphRule` в БД, роутер `/api/v1/graph-rules/` и сид-скрипты. 
   - **(СДЕЛАНО)** Интеграция правил в `agent_crew.py`: передача этих инструкций в системный промпт DeepSeek на этапе `_compile_declaration` для корректировки финального JSON.
   - **(ЗАДАЧА)** Создание UI в админке (`AdminGraphRulesPage.tsx`) для редактирования текстовых правил пользователями (например, *"Если поставщик ZED Group, всегда ставь условия поставки EXW"*).

3. **Связывание документов (Side-by-Side Viewer)** - *В ОЧЕРЕДИ*
   - Доработка UI `DeclarationEditPage.tsx`.
   - Создание сплит-экрана: слева — форма декларации, справа — встроенный PDF-вьювер (iframe или react-pdf).
   - Возможность кликнуть на документ в списке прикрепленных файлов и сразу увидеть его содержимое для визуальной сверки.

## Фаза 2: Trade Compliance & Финансовый контроль
*Цель: Снижение рисков штрафов для клиентов и брокеров.*

1. **Модуль Санкционного комплаенса (Trade Compliance)** - *В ОЧЕРЕДИ*
   - Создание микросервиса `compliance-service`.
   - Интеграция с открытыми API санкционных списков (OFAC, EU Sanctions Map, списки РФ).
   - Автоматическая проверка:
     - Контрагентов (Отправитель, Получатель) по названиям и ИНН/Tax ID.
     - Кодов ТН ВЭД на предмет товаров двойного назначения.
   - Вывод предупреждений (Alerts) в UI декларации.

2. **Валютный контроль (УНК и Платёжки)** - *В ОЧЕРЕДИ*
   - Расширение парсеров в `ai-service`:
     - `unk_parser.py` для парсинга Ведомостей банковского контроля (извлечение номера УНК, банка, сумм).
     - `payment_order_parser.py` для парсинга платежных поручений.
   - Сверка данных: сумма инвойса vs сумма по УНК vs сумма по платежкам.
   - Вывод баланса по контракту в интерфейсе.

## Фаза 3: Цифровой Логист (Supply Chain)
*Цель: Расширение воронки продаж за счет логистических услуг.*

1. **ИИ-Тарификатор (Smart Quoting)** - *В ОЧЕРЕДИ*
   - Парсер ставок морских линий и ж/д операторов из Excel/PDF в структурированную базу тарифов.
   - Интерфейс запроса ставки: ввод габаритов, веса, маршрута -> выдача 3-х вариантов логистики (быстро, дешево, оптимально).

2. **Оптимизатор загрузки (Load Planner)** - *В ОЧЕРЕДИ*
   - Использование данных из Packing List (вес, объем, габариты коробок).
   - Расчет необходимого количества контейнеров (20', 40'HQ) или фур.
   - Визуализация (опционально) или просто текстовый расчет утилизации объема.

3. **Генератор транспортных документов** - *В ОЧЕРЕДИ*
   - На основе данных инвойса и пакинга — автогенерация CMR, AWB, коносаментов в формате PDF (через WeasyPrint или ReportLab).

## Фаза 4: Интеграции и SaaS-метрики
*Цель: Встраивание в существующие IT-ландшафты клиентов и монетизация.*

1. **Экспорт в Альта-ГТД / СТМ** - *В ОЧЕРЕДИ*
   - Развитие `integration-service`.
   - Создание системы шаблонов (Jinja2) для маппинга внутреннего JSON декларации во внутренние XML-форматы популярных таможенных программ.
   - Скачивание готового файла для импорта в Альта-ГТД.

2. **Интеграция с 1С (Адаптер)** - *В ОЧЕРЕДИ*
   - Генерация XML/JSON в формате EnterpriseData для 1С:ERP / 1С:Бухгалтерия.
   - Автоматическая передача данных о выпущенных товарах (Поступление товаров и услуг) клиенту.

3. **Cost Governance (Биллинг ИИ)** - *В ОЧЕРЕДИ*
   - Подсчет токенов (input/output) от DeepSeek на каждую декларацию.
   - Сохранение стоимости обработки в БД.
   - Дашборд для админа с расчетом юнит-экономики (сколько стоит обработка 1 декларации в центах).

4. **Telegram-бот для клиентов (на базе OpenClaw или aiogram)** - *В ОЧЕРЕДИ*
   - Бот для клиентов брокера.
   - Функции: отправка документов в парсинг, запрос статуса декларации, уведомления о выпуске, запрос недостающих данных (например, "Укажите вес брутто для товара X").
