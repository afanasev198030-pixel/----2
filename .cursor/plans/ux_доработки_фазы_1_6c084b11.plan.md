# Доработки UX Фазы 1

Из 9 проверенных пунктов 7 не реализованы. Ниже план по каждому.

## 1. BrokerDashboard: кликабельные метрики с фильтрами

Карточки "Выпущено" и "Отклонено" не кликабельны. Нужно:

- Добавить `onClick` на все 4 карточки с навигацией `navigate('/declarations?status=...')`
- В `DeclarationsListPage` читать `?status=` из URL и автоматически устанавливать `statusFilter`
- Файлы: `BrokerDashboard.tsx`, `DeclarationsListPage.tsx`

## 2. Уведомления (колокольчик)

- Добавить иконку колокольчика в `AppLayout.tsx` рядом с аватаром
- Простой dropdown с последними 5 событиями из `declaration_logs` (статус изменён, документ загружен)
- Новый endpoint `GET /api/v1/notifications/recent` в core-api — последние 10 логов текущего пользователя
- Файлы: `AppLayout.tsx`, новый `services/core-api/app/routers/notifications.py`

## 3. Автосохранение черновиков

- В `DeclarationEditPage.tsx` добавить `useEffect` с debounce 5 секунд на `watchedValues`
- При изменении формы — автосохранение через `updateDeclaration(id, data)` + маленький индикатор "Сохранено" / "Сохранение..."
- Только для статуса `draft`
- Файл: `DeclarationEditPage.tsx`

## 4. Экспорт таблиц в Excel

- Добавить кнопку "Excel" на `DeclarationsListPage` и `ClientsListPage`
- Использовать клиентскую генерацию CSV (без backend) — простой `blob` download
- Файлы: `DeclarationsListPage.tsx`, `ClientsListPage.tsx`

## 5. Профиль пользователя

- Новая страница `ProfilePage.tsx` — смена пароля, редактирование имени
- Новый endpoint `PUT /api/v1/auth/profile` (full_name, password) в `auth.py`
- В `AppLayout.tsx` клик на аватар → переход на `/profile`
- Роут `/profile` в `App.tsx`
- Файлы: новый `ProfilePage.tsx`, `auth.py`, `AppLayout.tsx`, `App.tsx`

## 6. Тёмная/светлая тема

- В `AppLayout.tsx` добавить иконку солнце/луна рядом с профилем
- Хранить выбор в `localStorage('theme')`

- Файлы: `index.tsx`, `AppLayout.tsx`, возможно `theme.ts`
