# DeepSeek + UX улучшения: 4 этапа

## Этап 0: DeepSeek как приоритетная LLM (ПЕРВЫЙ ПРИОРИТЕТ)

DeepSeek API полностью совместим с OpenAI SDK — тот же формат `openai` Python-пакета, просто другой `base_url`. Не нужно менять библиотеки, только конфигурацию.

Источник: [awesome-deepseek-integration](https://github.com/deepseek-ai/awesome-deepseek-integration)

### 0.1 Поддержка multi-provider в ai-service config

Изменить `services/ai-service/app/config.py`:

- Добавить `LLM_PROVIDER` env var: `deepseek` (default), `openai`, `custom`
- Добавить `LLM_BASE_URL` env var: default `https://api.deepseek.com` для DeepSeek
- Добавить `LLM_API_KEY` env var (замена OPENAI_API_KEY, но с обратной совместимостью)
- Добавить `LLM_MODEL` env var: default `deepseek-chat` (DeepSeek V3), fallback `gpt-4o`
- Добавить `LLM_REASONING_MODEL`: default `deepseek-reasoner` (DeepSeek R1)
- Добавить `EMBED_MODEL`: default `text-embedding-3-small` (OpenAI) или `deepseek` embedding model
- `has_openai` property переименовать в `has_llm`

### 0.2 Обновить все OpenAI вызовы → multi-provider

Все файлы используют `openai.OpenAI(api_key=...)`. Нужно добавить `base_url`:

```python
# Было:
client = openai.OpenAI(api_key=settings.OPENAI_API_KEY)

# Стало:
client = openai.OpenAI(
    api_key=settings.LLM_API_KEY,
    base_url=settings.LLM_BASE_URL,  # https://api.deepseek.com для DeepSeek
)
```

Файлы для изменения (6 штук):

- `services/ai-service/app/services/dspy_modules.py` — GPT-4o fallback (line 297)
- `services/ai-service/app/services/invoice_parser.py` — `_llm_enrich` (line 756)
- `services/ai-service/app/services/packing_parser.py` — `_llm_parse_pl` (line 29)
- `services/ai-service/app/services/index_manager.py` — embeddings (line 106)
- `services/ai-service/app/main.py` — `/api/v1/ai/configure` handler
- `services/core-api/app/routers/settings.py` — key validation

### 0.3 Обновить docker-compose.yml и .env

```env
# .env — новые переменные
LLM_PROVIDER=deepseek
LLM_BASE_URL=https://api.deepseek.com
LLM_API_KEY=sk-xxx
LLM_MODEL=deepseek-chat
LLM_REASONING_MODEL=deepseek-reasoner
```

docker-compose.yml — ai-service environment:

```yaml
LLM_PROVIDER: ${LLM_PROVIDER:-deepseek}
LLM_BASE_URL: ${LLM_BASE_URL:-https://api.deepseek.com}
LLM_API_KEY: ${LLM_API_KEY:-${OPENAI_API_KEY}}
LLM_MODEL: ${LLM_MODEL:-deepseek-chat}
```

### 0.4 Обновить UI настроек

`SettingsPage.tsx`:

- Селектор провайдера: DeepSeek / OpenAI / Custom
- Поле API ключа (универсальное)
- Поле Base URL (для custom)
- Селектор модели: deepseek-chat, deepseek-reasoner, gpt-4o, gpt-4o-mini
- Ссылка на [platform.deepseek.com](https://platform.deepseek.com) для получения ключа

### 0.5 Поддержка Excel/XLSX файлов (парсинг документов)

Сейчас `ocr_service.py` поддерживает только PDF и изображения. Excel-файлы (упаковочные листы, спецификации) игнорируются.

Пример реального файла: `784-82985781 PL.xlsx` (13KB, упаковочный лист в Excel).

**Изменения:**

1. Добавить `openpyxl>=3.1.0` в `services/ai-service/requirements.txt`
2. В `services/ai-service/app/services/ocr_service.py`:
  - Новая функция `extract_text_from_excel(file_bytes, filename) -> str`
  - Читает все листы через `openpyxl.load_workbook(BytesIO(file_bytes))`
  - Для каждой строки: объединяет ячейки через табуляцию
  - Возвращает текстовое представление таблицы
  - В `extract_text()` добавить ветку: `if filename_lower.endswith(('.xlsx', '.xls')):`
3. В `services/file-service/app/routes.py`:
  - Добавить MIME types: `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`, `application/vnd.ms-excel`
4. В `services/ai-service/app/services/agent_crew.py`:
  - В `detect_doc_type()` добавить: `if filename_lower.endswith('.xlsx'):` → определять как invoice/packing_list по содержимому
5. В `frontend/src/components/DocumentUploadPanel.tsx`:
  - Добавить `.xlsx, .xls` в accept аттрибут input файла

### 0.6 Embeddings — стратегия (для RAG)

DeepSeek пока не имеет собственного embedding API. Варианты:

- Использовать ChromaDB default embeddings (onnxruntime, бесплатно, offline) — уже работает
- Использовать OpenAI `text-embedding-3-small` только для embeddings (отдельный ключ)
- В config: `EMBED_PROVIDER=local` (default) или `EMBED_PROVIDER=openai`

### 0.6 Обратная совместимость

- Если `LLM_API_KEY` не задан, fallback на `OPENAI_API_KEY`
- Если `LLM_BASE_URL` не задан и провайдер `openai` — использовать стандартный URL OpenAI
- Все существующие `.env` файлы продолжают работать без изменений
